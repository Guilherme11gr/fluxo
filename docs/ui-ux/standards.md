# UI Standards

## Tema

- dark mode e o padrao real do app
- use classes semanticas do tema (`bg-background`, `text-foreground`, `border-border`, `bg-card`)
- evite cores hardcoded quando existir token equivalente

## Tipografia E Componentes

- interface usa `Inter`
- dados tecnicos e IDs usam `JetBrains Mono`
- prefira componentes do sistema atual (`components/ui` + Tailwind)

## Layout

- mobile first
- evite overflow horizontal
- use grids simples e progressivos (`grid-cols-1`, `md:`, `lg:`)

## Interacao

- feedback de loading claro
- empty states com CTA
- hover e transicoes sutis; sem brilho ou excesso visual

## Regra Pratica

- preserve o visual existente do app em vez de inventar uma linguagem nova por tela
